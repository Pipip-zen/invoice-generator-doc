Overview

About This Project

Invoice Tracker adalah aplikasi web full-featured untuk mengelola invoice bisnis secara profesional. Dibangun dengan React dan Tailwind CSS di frontend, dengan Supabase sebagai backend serverless yang menyediakan database dan real-time capabilities.

Proyek ini mendemonstrasikan kemampuan dalam membangun aplikasi CRUD yang production-ready, mulai dari desain UI yang clean hingga integrasi dengan external service seperti Supabase, serta fitur export PDF yang memanfaatkan browser API.

Seluruh proses development diselesaikan dalam satu hari, mencakup perencanaan, implementasi, testing, dan deployment ke Vercel.

Tech Stack

Technologies Used

⚛️

React + Vite

Frontend Framework

Component-based architecture yang memudahkan reusability dan maintainability. Vite dipilih untuk development experience yang jauh lebih cepat dibanding CRA.

🎨

Tailwind CSS

Styling

Utility-first CSS yang memungkinkan styling cepat tanpa meninggalkan HTML. Konsistensi desain terjaga karena menggunakan design system bawaan Tailwind.

🗄️

Supabase

Backend as a Service

PostgreSQL database dengan REST API otomatis. Dipilih karena setup cepat, gratis, dan tidak perlu menulis backend code.

📈

Recharts

Data Visualization

Library chart berbasis React yang ringan dan mudah dikustomisasi. Digunakan untuk menampilkan revenue trend dan breakdown per kategori.

📝

React Hook Form

Form Management

Validasi form yang performant dengan minimal re-render. Dikombinasikan dengan Zod untuk type-safe schema validation.

🚀

Vercel

Deployment

Platform deployment yang terintegrasi langsung dengan GitHub. Auto-deploy setiap push ke main branch, dengan free SSL dan CDN global.

Learning

Challenges & Solutions

⚠️

Dynamic Line Items Calculation

Challenge

Mengelola state untuk dynamic line items di form invoice cukup tricky — setiap perubahan quantity atau unit price harus langsung recalculate amount per item dan total keseluruhan secara real-time.

Solution

Menggunakan useFieldArray dari React Hook Form untuk mengelola array items, dikombinasikan dengan useWatch untuk memonitor perubahan nilai dan trigger recalculation secara otomatis.

⚠️

PDF Export Layout

Challenge

html2canvas tidak selalu merender Tailwind classes dengan benar, terutama untuk elemen yang menggunakan dynamic classes atau gradients. Hasilnya PDF terlihat berbeda dari tampilan di browser.

Solution

Membuat komponen invoice terpisah khusus untuk print/export dengan inline styles, dan menggunakan hidden div yang dirender saat export dipicu. Hasilnya konsisten di semua browser.

💡

What I Learned

Proyek ini mengajarkan saya pentingnya perencanaan data structure sebelum mulai coding — keputusan schema database di awal sangat mempengaruhi kompleksitas query dan state management di frontend. Saya juga semakin yakin bahwa Supabase adalah pilihan tepat untuk proyek yang membutuhkan backend cepat tanpa overhead. Dan yang terpenting, saya belajar bahwa pengalaman pengguna dalam mengisi form yang kompleks seperti invoice sangat bergantung pada immediate feedback — auto-calculation, validation inline, dan toast notification membuat perbedaan besar.
